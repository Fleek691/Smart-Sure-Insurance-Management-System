SmartSure Insurance Management System
Project Requirement Document (Microservices)
Angular • ASP.NET Core Microservices (4) • Ocelot API Gateway • SQL Server • EF Core Code-First • JWT

A) Application Overview
SmartSure Insurance Management System is a web-based platform designed to digitize the complete insurance lifecycle. Customers can register, purchase insurance policies, calculate premiums, upload documents, and initiate claims using guided workflows. Admin users manage policy products, verify claim documents, approve or reject claims, and generate operational reports. The application is built using Angular and ASP.NET Core microservices, with Ocelot API Gateway enabling secure and scalable service orchestration.

B) Objective
• Digitize policy purchase and claim settlement processes
• Reduce manual processing using workflow-driven status transitions
• Enforce strong security using JWT and role-based authorization
• Demonstrate enterprise-grade microservices architecture
• Provide reporting and dashboards for operational visibility

C) Roles & Access Control
Roles
Customer (Policy Holder)
• Register/Login
• Buy insurance policies
• View policies and documents
• Initiate and track claims
Admin
• Manage insurance products
• Review and verify claim documents
• Approve or reject claims
• Generate reports and manage users
Security
• Angular Route Guards for /customer/* and /admin/*
• JWT Bearer authentication
• [Authorize(Roles="ADMIN")] on admin APIs
• All requests routed via Ocelot Gateway

G) Microservices System Architecture
Architecture Overview
Angular UI communicates only with Ocelot API Gateway.
Ocelot routes requests to four independent microservices, each owning its data.


Microservices
Identity / Auth Service
• Authentication, JWT, roles
Policy Management Service
• Insurance products, policies, premiums
Claims Service
• Claims, documents, verification
Admin / Reporting Service
• Approvals, dashboards, reports
